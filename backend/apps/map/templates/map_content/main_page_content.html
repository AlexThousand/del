Описание логики работы страницы патрулирования

Город разделён на несколько зон патрулирования. Каждая зона обслуживается определённым набором патрулей.

Основной функционал:

  • Пользователь выбирает на карте место происшествия, кликая по карте.
  • Приложение рассчитывает оптимальные маршруты движения от каждого патруля до места происшествия.
  • Для построения маршрутов используется Open Source Routing Machine (OSRM) — локальный контейнер, который строит оптимальную траекторию исключительно по дорогам общего пользования.

Алгоритм выбора патруля для реагирования:

  1. Из всех патрулей выбирается самый близкий к месту происшествия, который принадлежит зоне реагирования.
  2. Если в зоне реагирования нет доступных патрулей, выбирается просто самый близкий патруль по расстоянию.

Таким образом, система обеспечивает быстрое и корректное распределение сил для реагирования, минимизируя время прибытия на место.
